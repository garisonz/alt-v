# ALT-V Project: Technical Concepts & Architecture Log

## 1. System Architecture
Utilized a Three Tier Architecture hosted on **WSL (Linux)** but orchestrated via **Docker**.

### The Stack
- **Frontend:** React + Vite (Running on Node.js)
- **Backend:** Python FastAPI (Running in a `venv`)
- **Database:** PostgreSQL (Running in a Docker Container)

## 6. Command List
**Start Database:** `docker-compose up -d`
**Stop Database:** `docker-compose down`
**Run Backend:** `uvicorn app.main:app --reload` (Must be in `backend/` folder)
**Run Frontend:** `npm run dev` (Must be in `frontend/` folder)

## 7. File Structure

alt-v/
├── .gitignore               # The "Bouncer" (Keeps junk out of GitHub)
├── docker-compose.yml       # The "Landlord" (Rents/Runs the Database)
├── LEARNING_LOG.md          # The "Brain" (Your personal notes)
│
├── backend/                 # THE ENGINE (Python/FastAPI)
│   ├── .env                 # Secrets (Passwords/Keys) - NEVER COMMIT THIS
│   ├── requirements.txt     # Ingredient List (Libraries needed to run this)
│   ├── venv/                # The Sandbox (Isolated Python environment)
│   └── app/                 # Your Source Code
│       ├── main.py          # The "Front Desk" (Entry point for the API)
│       ├── api/             # (Future) Endpoints (Where users ask for data)
│       ├── core/            # Configuration (Settings & Security)
│       ├── db/              # The Vault (Database connection & Tables)
│       └── etl/             # The Factory (Scripts that fetch/clean data)
│
├── frontend/                # THE FACE (React/Vite)
│   ├── package.json         # Ingredient List (JS libraries)
│   ├── vite.config.js       # Build Settings
│   ├── node_modules/        # The Heavy Libraries (Auto-generated)
│   └── src/                 # Your UI Code
│       ├── App.jsx          # The Main Component
│       └── main.jsx         # The Mounting Point
│
└── postgres_data/           # THE STORAGE (Auto-generated by Docker)
